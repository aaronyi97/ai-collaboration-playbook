# AI 接棒卡模板

当新 AI 对话需要接着旧任务做，但看不到旧对话时，用这份模板。

## 1. 目标

我们最终要完成什么？

```text
<最终目标>
```

## 2. 当前状态

- 当前阶段：`<规划 / 执行 / 排错 / 审查 / 验证 / 发布>`
- 当前分支或工作目录：`<如适用，写 branch/path>`
- 当前真实状态：`<现在已经到哪一步>`

## 3. 已完成

只写真正完成的事。

- `<已完成事项>`  
  证据：`<文件 / commit / 命令 / 链接>`

## 4. 关键决定

- `<决定>`  
  原因：`<为什么这么定>`  
  证据：`<如有，写来源>`

## 5. 改动文件

| 文件 | 改动 | 状态 |
|---|---|---|
| `<路径>` | `<改了什么>` | `<已完成 / 待处理 / 未验证>` |

## 6. 废弃路线

这些方案已经试过、否决或替换。除非用户明确要求，不要重复走。

- `<废弃路线>`  
  原因：`<为什么废弃>`

## 7. 风险与未知

### 已验证事实

- `<事实>`  
  证据：`<来源>`

### 推断

- `<推断>`  
  置信度：`<高 / 中 / 低>`  
  理由：`<为什么这么判断>`

### 未确认

- `<未知项>`  
  怎么确认：`<下一步检查方式>`

## 8. 下一步

新 AI 对话第一步应该做：

```text
<具体第一动作>
```

## 9. 先读文件

行动前先读：

1. `<文件/链接>` — `<为什么重要>`
2. `<文件/链接>` — `<为什么重要>`

## 10. 新 AI 接手 Prompt（可选）

这是配合接棒卡一起发给新对话的开场指令。它的作用不是补充更多内容，而是防止新 AI 看完接棒卡后直接乱执行。

如果你已经在简版接棒卡里写了“新 AI 先复述目标、状态、风险和第一步，用户确认后再执行”，这一节可以不单独展开。

需要更稳时，把下面这段和接棒卡一起发给新 AI：

```text
请先阅读这份 AI 接棒卡。
你看不到旧对话，不要假设你知道前文。
执行前先回复：
1. 目标是什么；
2. 当前状态是什么；
3. 主要风险是什么；
4. 你第一步要做什么；
5. 你不会做什么。
如果有任何不清楚，先问，不要直接执行。
```
