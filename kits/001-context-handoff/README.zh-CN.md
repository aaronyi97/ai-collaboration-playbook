# Kit 001：AI 接棒卡

[English README](README.md)

## 解决什么问题

你和一个 AI 对话聊了很久。它已经知道项目目标、当前状态、改过哪些文件、哪些方案被否决、下一步该做什么。

然后你开了一个新对话。

新 AI 又从零开始问你：

> 这个项目是干什么的？现在做到哪一步了？

这不只是记忆问题，而是交接问题。

## 解决方案

用一张 **AI 接棒卡**。

它是一份短的结构化交接文档，让新 AI 即使看不到旧对话，也能继续任务。

接棒卡会记录：

- 目标；
- 当前状态；
- 已完成；
- 已做决定；
- 改动文件；
- 废弃路线；
- 风险；
- 下一步；
- 先读文件。

## 从这里开始

如果你用的是 OpenClaw、Cursor、Claude Code、Codex 或其他 AI 编程工具：

[复制中文视频置顶评论版](PINNED.zh-CN.md)

打开后复制代码块里的整段文字，不要只把这个 GitHub 链接发给 AI。

这个 Prompt 默认是**直接交接模式**：它会从当前对话和项目上下文里直接生成一份简版接棒表，放进代码块方便复制。信息不足的地方会标成“未确认”，不会先让用户填一长串问题。

简版表后只问一句：是否需要继续生成详细版接棒卡。

## 什么时候用

适合这些情况：

- 对话太长，准备开新对话；
- 从一个 AI 工具换到另一个 AI 工具；
- 从一个模型换到另一个模型；
- 让另一个 AI 接着做同一个任务；
- AI 编程任务里已经有很多决定和文件改动；
- 新 AI 必须知道哪些旧路不要再走。

## 轻量流程

```text
旧对话
-> 直接生成简版 AI 接棒卡
-> 如有需要再生成详细版
-> 新对话读取接棒卡
-> 新对话先复述目标、风险、下一步
-> 用户确认
-> 新对话继续执行
```

## 不要怎么做

不要把整段旧聊天记录复制给新 AI。

原始聊天记录里通常混着：

- 废话；
- 试错；
- 旧方案；
- 后来被推翻的判断；
- 已经过期的工具输出。

接棒卡的作用，就是保留有用上下文，过滤噪音。

## 示例场景

- 在新 AI 对话里继续编程任务；
- 从 ChatGPT 切到 AI 编程工具；
- 从一个 coding agent 切到另一个；
- 把项目状态交给队友；
- 在高风险重构前保存一个检查点。
